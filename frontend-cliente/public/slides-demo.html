O Problema → A Solução 02 / 12

❌ Processo Atual (Presencial)

  • Cliente precisa ir à agência pessoalmente
  • Preenchimento manual de formulários em papel
  • Filas, espera e múltiplas visitas necessárias
  • Risco de perda de documentos físicos
  • Sem rastreabilidade do status da solicitação
  • Sem conformidade automática com o normativo
  • Dependência total de horário bancário
  • Arquivamento manual em pastas físicas

✅ EncerraDigital (100% Digital)

  • Solicitação online em qualquer hora e lugar
  • Formulário guiado multi-etapa ou chatbot interativo
  • Protocolo gerado automaticamente, rastreável
  • Geração automática do Termo BRF-3303-40-64
  • Assinatura digital via gov.br / ICP-Brasil
  • Dados cifrados AES-256-CBC em trânsito e repouso
  • Painel do operador com auditoria e LGPD
  • Status em tempo real sem expor dados sensíveis

Segurança & LGPD — OWASP ASVS 09 / 12

🔐 Segurança Implementada

🔑
AES-256-CBC — Dados em Repouso
titularNome, numeroConta, emailCliente e contaTransferencia criptografados no banco. Chave via variável de ambiente.
🛡️
Headers HTTP de Segurança
CSP, HSTS, X-Frame-Options: DENY, Referrer-Policy, Permissions-Policy via Helmet + next.config.mjs.
📁
Upload Seguro de Arquivos
Validação magic bytes %PDF, limite 10 MB, armazenamento fora do diretório público, hash SHA-256 por arquivo.
🚫
Sem Exposição de Stack Trace
API nunca retorna mensagens de erro internas. IP mascarado em logs (10.x.x.xxx). Dados pessoais nunca em logs.

⚖️ Conformidade LGPD

📋
Aceite de Termos Rastreável
Timestamp + versão do documento registrados em cada solicitação. Base legal documentada.
🎯
Coleta Mínima de Dados
Apenas os dados necessários para o processo de encerramento são coletados, conforme princípio da necessidade.
📊
Log de Auditoria Obrigatório
Todos os acessos a dados pessoais no módulo interno são logados com operador, id e timestamp.
👁️
Status Sem Exposição de Dados
Consulta pública de status retorna apenas fase da solicitação — sem nome, CPF ou dados bancários.
